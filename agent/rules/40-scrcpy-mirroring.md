# Scrcpy 屏幕投屏规范

1. **投屏物理状态控制与同步**
   - **投屏时关闭设备屏幕**：允许用户在投屏期间关闭 Android 设备的物理屏幕以节省电量。控制功能通过 ADB 发送特定的物理屏幕显隐命令（如通过控制接口或特定注入命令）。
   - **状态还原与清理**：在投屏结束时（主动关闭、拔出数据线或异常断开），物理屏幕的状态必须还原为亮起（即重置物理屏幕状态，如重置 `screenPowerOffProvider` 为 `false`），且 UI 表现层状态必须与之同步重置。
   - 投屏服务销毁或停止（例如调用 `toggleMirroring` 或 `forceStop`）时，后台必须主动回收投屏端口和进程。

2. **鼠标与触摸手势映射**
   - 投屏区域内的操作应支持：
     - **鼠标左键拖拽/滑动**：转换为触摸屏的 Drag/Slide 事件并实时发送至设备。
     - **右键返回**：鼠标右键点击默认映射为 Android 系统的 `BACK` 按键。
     - **中键返回主页**：鼠标中键点击默认映射为 Android 系统的 `HOME` 按键。
   - 手势坐标转换：在向底层 scrcpy 模块发送坐标前，必须根据投屏 Widget 的当前缩放比例以及 Android 设备的物理分辨率进行精确的坐标映射，防止点击偏移。

3. **键盘输入重定向**
   - 支持将电脑键盘的物理输入重定向转发给已连接的 Android 设备。
   - 当投屏窗口/区域处于聚焦状态时，拦截本地键盘事件（如 `Focus`、`KeyEvent` 监听），转换成 Android 键码并注入到设备中。当焦点切出时，停止键盘事件捕获。
   - 需要对快捷键进行特殊处理，确保类似 Cmd+C / Cmd+V 等操作在本地系统与设备端有清晰的职责区分。
