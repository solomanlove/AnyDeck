# Skill: doc-test-impact-spec (测试与回归影响文档)

## 概述
本技能用于评估改动对系统的影响范围，为 QA/测试人员生成对应的手动及半自动回归路径，并提供模拟环境的 ADB 指令。

## 测试文档模板 (Markdown)

```markdown
# 变更测试影响分析：[功能名称]

## 1. 受影响的功能模块 (Impact Areas)
- **直接受影响范围**：[具体页面、Tab 或配置项]
- **间接/潜在影响范围**：[例如：全局多语言切换对已打开的子窗口标题的影响、物理屏幕关闭是否会导致设备端锁屏等]

## 2. 验证方案 (Verification Plan)

### A. 自动静态检查
- 命令：
  ```bash
  flutter analyze
  ```

### B. 核心功能正向用例
1. [用例 1]：描述操作步骤 -> 预期现象。
2. [用例 2]：描述操作步骤 -> 预期现象。

### C. 边界与异常测试
- [异常 1]：物理数据线拔出后，投屏进程是否正常结束、物理屏幕状态是否恢复、UI 能否安全回到未连接状态。
- [异常 2]：主窗口异常崩溃退出时，子窗口的生命周期回收情况。
- [异常 3]：当 adb 命令执行超时（大于 15 秒）时，UI 是否展示了超时提示，而不是一直处于菊花 Loading 态。

### D. 原生及环境验证
- 在真实 Android 物理机及虚拟模拟器（AVD）上分别进行触控滑动、按键映射、应用列表读取的测试，记录表现差异。
```
